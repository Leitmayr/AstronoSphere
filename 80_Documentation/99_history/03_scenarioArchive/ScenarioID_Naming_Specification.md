
# ScenarioID Naming Specification

## Purpose

ScenarioIDs uniquely identify validation scenarios across the entire
AstronoSphere ecosystem.

The identifier must be:

• human readable  
• deterministic  
• globally unique  
• reproducible

---

# Two-Part Structure

A ScenarioID consists of two logical parts:

```
CORE_PART + FACTORY_PART
```

CORE_PART describes the **physical scenario**.  
FACTORY_PART describes **factory‑specific parameters**.

---

# Structure

```
[CORE]--[FACTORY]
```

Example:

```
HELIO-TDB-2451545-2451555-1D--EPH-HORIZONS-DE440
```

---

# Core Section

The core section describes the **physical observation scenario**.

Pattern:

```
[OBSERVER]-[TIMESCALE]-[STARTJD]-[STOPJD]-[STEP]
```

Example:

```
HELIO-TDB-2451545-2451555-1D
```

Components:

Observer:
HELIO
GEO
TOPO

TimeScale:
TT
TDB
UTC

Step:
1D
6H
15M

---

# Factory Section

The factory section describes how the **reference data was produced**.

Pattern:

```
[FACTORY]-[SOURCE]-[MODEL]
```

Example:

```
EPH-HORIZONS-DE440
```

Possible values:

Factory:

EPH → EphemerisFactory  
TIME → TimeFactory  
OBS → ObserverFactory

Source:

HORIZONS  
SPICE  
CUSTOM

Model:

DE440  
DE441  
VSOP87

---

# Full Example

```
HELIO-TDB-2451545-2451555-1D--EPH-HORIZONS-DE440
```

Meaning:

Heliocentric observer  
TDB time scale  
Start JD 2451545  
Stop JD 2451555  
Step 1 day  
Generated by EphemerisFactory  
Using JPL Horizons  
Reference model DE440

# Benefits

• deterministic naming  
• easy CLI filtering  
• human readability  
• stable identifiers across repositories

---

# CLI Filtering Examples

Retrieve all heliocentric scenarios:

```
HELIO*
```

Retrieve all DE440 scenarios:

```
*DE440
```

Retrieve EphemerisFactory runs:

```
*EPH*
```

---

# Governance Rules

1. The CORE part must remain stable.
2. Factory parameters must never alter the core scenario.
3. Factories must not reinterpret CORE semantics.
4. New factories may extend only the FACTORY section.

---

# Summary

ScenarioIDs provide a **stable linking key** connecting:

• scenario definition  
• reference datasets  
• factory runs  
• analysis reports  
• validation results

They are the **primary index of the AstronoSphere ecosystem**.
