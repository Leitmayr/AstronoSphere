# AstronoSphere Session Protocol

## Purpose

This protocol defines a lightweight operational framework to maintain
control, focus, and determinism during development sessions.

------------------------------------------------------------------------

# 1. Session Start (Mandatory)

Before any coding:

-   Define the goal (one sentence)
-   Identify the active dimension (ONLY ONE)
-   Define what remains frozen
-   Define validation strategy

------------------------------------------------------------------------

# 2. Implementation Mode (Engineer Mode)

Rules:

-   Change exactly ONE dimension
-   Do not modify frozen components
-   No side changes
-   Follow predefined validation

------------------------------------------------------------------------

# 3. Bug Mode (Scientist Mode)

Trigger: frustration or repeated failure

## Protocol

1.  Freeze system
2.  Define scope (max 2--3 possible causes)
3.  Define ONE hypothesis
4.  Define validation method

## Rule

Only ONE hypothesis at a time.

------------------------------------------------------------------------

# 4. Idea Mode (Architect Mode)

Trigger: new idea during work

## Protocol

Document and park:

-   Idea
-   Affected dimension
-   Target milestone

Do NOT implement immediately.

------------------------------------------------------------------------

# 5. Definition of Done

A task is complete ONLY if:

-   Run == LastRun
-   Golden samples validated
-   No unexplained deviations
-   System stable

------------------------------------------------------------------------

# 6. Core Rules

-   One dimension at a time
-   Validation before implementation
-   Spec over intuition
-   Preparation is mandatory

------------------------------------------------------------------------

# 7. Mindset

Engineer → build\
Scientist → validate\
Architect → decide

Switch consciously between modes.

------------------------------------------------------------------------

# End of Document
