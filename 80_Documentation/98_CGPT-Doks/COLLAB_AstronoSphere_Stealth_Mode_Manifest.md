# AstronoSphere -- Stealth Mode Manifest (Level 1)

## Purpose

This manifest defines the operational discipline for achieving
**Scientific Closure (Level 1)**.

Stealth Mode ensures maximum focus, zero distraction, and uncompromising
scientific integrity.

------------------------------------------------------------------------

# 1. Core Principle

> **Everything that does not directly improve scientific validation does
> not exist.**

------------------------------------------------------------------------

# 2. Active Scope (Allowed Work)

Only the following are allowed:

-   L0 → L2 Physics (Astronometria)
-   GroundTruth validation (Horizons)
-   Deterministic pipeline stability
-   Mesh expansion & system behavior analysis
-   Astronolysis (analysis + seed generation)
-   Validation infrastructure (compare, hashes, golden samples)

------------------------------------------------------------------------

# 3. Forbidden Scope (Strict)

The following are explicitly **NOT allowed**:

-   GUI / Visualization
-   INDI / Telescope integration
-   API development
-   Chebyshev or alternative engines
-   New instruments beyond defined scope
-   Any UX-related work

> Even "small" changes are forbidden.

------------------------------------------------------------------------

# 4. Dimensional Discipline

> **At any time, exactly ONE dimension may change.**

Examples: - Physics Level (L0 → L1) - Scenario Coverage (144 → Mesh)

All other dimensions remain frozen.

------------------------------------------------------------------------

# 5. Validation First

> No change without predefined validation.

Every task must define:

-   Hypothesis
-   Expected behavior
-   Validation method

------------------------------------------------------------------------

# 6. Definition of Done

A task is complete only if:

-   Run == LastRun (binary identical)
-   Golden samples validated
-   No unexplained deviations
-   Behavior understood and explainable

------------------------------------------------------------------------

# 7. Bug Protocol

When frustration occurs:

1.  Freeze system
2.  Define scope (max 2--3 causes)
3.  Test ONE hypothesis
4.  Validate clearly

> No multi-variable debugging.

------------------------------------------------------------------------

# 8. Idea Handling

All new ideas must be:

-   documented
-   assigned to future milestone

> No implementation during Stealth Mode.

------------------------------------------------------------------------

# 9. Mindset

You are not building features.

> You are validating a physical model.

Roles: - Engineer → implement - Scientist → validate - Architect →
decide (but NOT execute now)

------------------------------------------------------------------------

# 10. Exit Condition

Stealth Mode ends when:

-   L0--L2 are fully validated
-   System behavior is understood
-   Scientific trust is established

------------------------------------------------------------------------

# End of Manifest
