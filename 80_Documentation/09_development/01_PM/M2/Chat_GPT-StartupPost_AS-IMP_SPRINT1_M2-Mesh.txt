## Fundamental Rule (NON-NEGOTIABLE)

All values MUST be computed algorithmically.

```text
ValidationMesh = {
    t ∈ SimulationMesh
    AND t ∈ ProviderRange(planet)
}
```

DO NOT:

* use any precomputed JD_Start / JD_Stop
* copy values from documentation
* manually adjust results

### 2. Validation Mesh Generation

Generate:

| Epoch  | Name |
| ------ | ---- |
| Epoch1 | MVH1 |
| Epoch2 | MVH2 |
| Epoch3 | MVH3 |

For each planet:

* determine ProviderRange (Horizons)
* intersect with SimulationMesh

---

## Mathematical Definitions (STRICT)

### JD_Start

```text
JD_Start = first t ∈ SimulationMesh WHERE t ≥ ProviderMin
```

---

### JD_Stop

```text
JD_Stop = last t ∈ SimulationMesh WHERE t ≤ ProviderMax
```

---

### Step

```text
Step = fixed per Epoch (30 / 211 / 809 days)
```

---

### Sample Rule

```text
N = number of steps
N < 2000
numberSamples = N + 1
```

---

## Output Requirements

For each generated seed:

* strictly follow structure in MeshGenerator_ExampleSeed.json
* no deviations in naming
* no missing fields
* no additional fields

Important details:

* TimeScale = "TDB"
* Observer = Heliocentric / Sun
* Frame = HelioEcliptic / J2000
* Author = "MeshGenerator"

---

### ResultID Rules

```text
StartID = integer part of JD_Start (truncate, not round)
StopID  = integer part of JD_Stop (truncate, not round)
```

## Validation Rules (must be implemented)

For every JD:

### 1. Grid Alignment

```text
(JD - MeshStart) % Step == 0
```

---

### 2. Range Check

```text
JD_Start ≥ ProviderMin
JD_Stop  ≤ ProviderMax
```

---

### 3. Maximality

```text
JD_Stop + Step > ProviderMax
```
