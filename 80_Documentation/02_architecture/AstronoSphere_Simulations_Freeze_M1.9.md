# AstronoSphere – Simulations & Naming Freeze (M1.9)

## 1. Purpose

This document defines the **Simulation layer** of AstronoSphere.

Simulations are generated by:

- Astronometria (primary engine)
- future engines (e.g., Miriade)

They represent:

> Computed results of the same experiments defined in the ObservationCatalog

---

## 2. Core Principle

Simulations MUST mirror GroundTruth structure.

Goal:

- Direct comparability
- Deterministic validation
- Minimal tooling complexity

---

## 3. Folder Structure (Freeze)

```
AstronoData/
  04_Simulations/
    Ephemeris/
      Astronometria/
        Run/
        LastRun/
        Baseline/
```

### Rules

- Domain-first (Ephemeris)
- Provider-second (Astronometria)
- Same structure as GroundTruth
- Additional engines can be added in parallel

---

## 4. Run / LastRun / Baseline

### Run
- Current execution output
- Fully overwritten each run

### LastRun
- Previous execution snapshot
- Used for regression comparison

### Baseline
- Scientific reference state
- Corresponds to a Git baseline
- Only changes intentionally

---

## 5. SimulationID

SimulationID defines the identity of simulation output.

Format:

```
<PROVIDER>-<INSTRUMENT>-<LEVEL>
```

Example:

```
ASTRONOMETRIA-VEC-L0
```

### Rules

- Provider mandatory
- Instrument mandatory (VEC, RADEC, ...)
- Level mandatory (L0, L1, ...)

---

## 6. Filename Convention

```
<HUMAN>__<SCENARIO>__<SIMULATION>.json
```

Example:

```
PLANET-MERCURY-INC__HELIO-J2000-TDB-2451545-2451546-1H__ASTRONOMETRIA-VEC-L0.json
```

---

## 7. Design Rationale

### 7.1 Symmetry with GroundTruth

GroundTruth:

```
HORIZONS-VEC-L0
```

Simulation:

```
ASTRONOMETRIA-VEC-L0
```

→ Enables direct comparison

---

### 7.2 Separation of Concerns

- Scenario → physics
- Simulation → computed result

---

### 7.3 No Redundancy

Domain (Ephemeris) is defined by folder.

Therefore NOT included in SimulationID.

---

## 8. Scientific Workflow

```
Seed → Experiment → Dataset → Simulation → Result
```

---

## 9. Freeze Decision (M1.9)

- Folder structure frozen
- SimulationID format frozen
- Filename convention frozen
- Baseline concept frozen
