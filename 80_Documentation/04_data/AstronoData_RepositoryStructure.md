# AstronoData Repository Structure

AstronoData stores scenarios, reference datasets, factory outputs, and
validation reports.

---

## Repository Layout

AstronoData
│
├ ObservationCatalog
│
├ ReferenceData
│
├ FactoryRuns
│
├ AccuracyReports
│
├ FactoryTools
│
└ OrchestrationTools


---

## ObservationCatalog

Contains scenario definitions.

Each scenario is defined as a JSON file.

Example:


ObservationCatalog/
	TS-A/
		TS-A_MARS_HELIOECL_L0_Q1.json


Scenarios define:

- ScenarioID
- Target
- Frame
- CorrectionLevel
- StartJD
- StopJD
- StepJD
- TimeScale

---

## ReferenceData

Scientifically validated datasets used by Astronometria tests.

ReferenceData is the **single source of scientific truth**.

Datasets are organized by scenario.

---

## FactoryRuns

Temporary datasets generated by factories.

Used for regression comparison and validation.

Structure:
FactoryRuns/
	EphemerisFactory/
		Run/
		LastRun/
		

---

## AccuracyReports

Statistical evaluation datasets used for quantitative validation of
Astronometria against reference data.
